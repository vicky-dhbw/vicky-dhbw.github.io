Introduction to Data Warehouse

A data warehouse is a large, centralized repository of data that is specifically designed to support business intelligence (BI) activities, data analysis, and reporting. It is a structured and integrated collection of data from various sources, such as databases, applications, and external systems. The primary purpose of a data warehouse is to provide a single, consolidated view of historical and current data that can be used for in-depth analysis and decision-making.


A data warehouse is a an OLAP (Online Analytical Processing) system which means it is designed for complex and ad-hoc querying and analysis of large volumes of historical data. OLTP (Online Transaction Processing) systems on the other hand are designed for real-time transactional processing, including tasks like inserting, updating, and deleting records. They are used for day-to-day operations like order processing and inventory management.

To simplify the management of multiple database servers on the host machine, we will employ Docker containers. These containers will serve as the foundation for our database setup. Our tool of choice for interacting with these containers will be Azure Data Studio. Before diving into the implementation, let's clarify our objectives. Our initial tasks involve the creation of two Microsoft SQL Server database containers. One of these will function as our data warehouse, while the other will serve as a typical database, which we'll refer to as the OLTP database. Following this, we'll proceed to generate sample tables within the OLTP database and populate them with data. Our next step will be establishing the OLTP database server as a linked server within the data warehouse database, enabling us to execute queries that extract data from the OLTP database.
